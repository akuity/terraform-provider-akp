# Terraform AKP Provider Version 0.14 Upgrade Guide

Version 0.14.0 of the AKP provider for Terraform introduces a `connectivity` attribute for selecting how Argo CD and Kargo instances, clusters, and agents are reached — over the public internet or a private AWS PrivateLink connection — and adds support for managing Kargo `ClusterConfig` resources declaratively through `kargo_resources`. This guide covers the upgrade from version 0.13.x to version 0.14.0.

## Provider Version Configuration

→ Before upgrading to version 0.14.0, upgrade to the most recent 0.13.x version (0.13.0) of the provider and ensure that
your environment successfully runs [`terraform plan`](https://www.terraform.io/docs/commands/plan.html). You should not
see changes you don't expect or deprecation notices. Also, back up your `.tf` and `.tfstate` files prior to starting the
upgrade process.

Use [version constraints when configuring Terraform providers](https://www.terraform.io/docs/configuration/providers.html#provider-versions).
If you are following that recommendation, update the version constraints in your Terraform configuration and
run [`terraform init -upgrade`](https://www.terraform.io/docs/commands/init.html) to download the new version.

For example, given this previous configuration:

```terraform
terraform {
  required_providers {
    akp = {
      source  = "akuity/akp"
      version = "~> 0.13.0"
    }
  }
}

provider "akp" {
  # Configuration options
}
```

Update to the latest 0.14.x version:

```terraform
terraform {
  required_providers {
    akp = {
      source  = "akuity/akp"
      version = "~> 0.14.0"
    }
  }
}

provider "akp" {
  # Configuration options
}
```

## New Features

### `connectivity`

Version 0.14.0 adds a `connectivity` attribute that selects how a resource is reached: `public` (over the internet) or `private` (over an AWS PrivateLink connection). It is available on the following surfaces:

- `akp_instance` — `spec.instance_spec.connectivity` (the Argo CD instance itself)
- `akp_instance` — `spec.instance_spec.cluster_customization_defaults.connectivity` (default applied to new Argo CD agents)
- `akp_cluster` — `spec.data.connectivity` (an individual Argo CD agent/cluster)
- `akp_kargo_instance` — `spec.kargo_instance_spec.connectivity` (the Kargo instance itself)
- `akp_kargo_instance` — `spec.kargo_instance_spec.agent_customization_defaults.connectivity` (default applied to new Kargo agents)
- `akp_kargo_agent` — `spec.data.connectivity` (an individual Kargo agent)

```terraform
resource "akp_instance" "example" {
  name = "argocd"
  argocd = {
    spec = {
      instance_spec = {
        # How the Argo CD instance is reached: "public" (internet) or "private" (AWS PrivateLink).
        connectivity = "private"
        # Default connectivity applied to newly created agents.
        cluster_customization_defaults = {
          connectivity = "private"
        }
      }
    }
  }
}
```

```terraform
resource "akp_cluster" "example" {
  instance_id = akp_instance.example.id
  name        = "test-cluster"
  spec = {
    data = {
      size = "small"
      # How this cluster's agent is reached.
      connectivity = "private"
    }
  }
}
```

`connectivity` accepts only `public` or `private`; any other value is rejected at plan time. The attribute is optional and can be updated in place — changing it does not force replacement. Because it is a computed attribute, it defaults to `public` when omitted, and existing configurations that never set it will not see a plan diff after upgrading. The `private` option requires an AWS PrivateLink connection to be provisioned for your organization.

### Managing Kargo `ClusterConfig`

Version 0.14.0 lets you manage a Kargo `ClusterConfig` — cluster-scoped configuration shared across all projects, such as webhook receivers — through the `kargo_resources` map of `akp_kargo_instance`. Provide the resource as a YAML document keyed by name, exactly like the other Kargo custom resources.

```terraform
resource "akp_kargo_instance" "example" {
  name = "test-kargo"

  kargo_resources = {
    "clusterconfig.yaml" = <<-EOT
apiVersion: kargo.akuity.io/v1alpha1
kind: ClusterConfig
metadata:
  name: cluster
spec:
  webhookReceivers:
  - name: github-wh
    github:
      secretRef:
        name: gh-webhook-secret
EOT
  }

  kargo = {
    # ...
  }
}
```

`ClusterConfig` joins the set of supported `kargo_resources` group-kinds (see the `kargo_resources` attribute documentation for the full list). Resources managed this way are exported back into Terraform state, so they participate in drift detection like any other managed Kargo resource.

## Apply the change

After updating the provider version, run `terraform plan` to verify no unintended changes were introduced. Both features are additive and opt-in: `connectivity` defaults to `public` when unset, and `ClusterConfig` support does not affect existing `kargo_resources` entries. If you notice any unexpected differences, please report them as issues.
