# Terraform AKP Provider Version 0.11 Upgrade Guide

Version 0.11.0 of the AKP provider for Terraform introduces cross-cluster Secret synchronization for both Argo CD and Kargo instances, and changes the behavior around deleting the default shard agent of a Kargo instance. This guide covers the upgrade from version 0.10.x to version 0.11.0.

## Provider Version Configuration

-> Before upgrading to version 0.11.0, upgrade to the most recent 0.10.x version (0.10.2) of the provider and ensure that
your environment successfully runs [`terraform plan`](https://www.terraform.io/docs/commands/plan.html). You should not
see changes you don't expect or deprecation notices. Also, back up your `.tf` and `.tfstate` files prior to starting the
upgrade process.

Use [version constraints when configuring Terraform providers](https://www.terraform.io/docs/configuration/providers.html#provider-versions).
If you are following that recommendation, update the version constraints in your Terraform configuration and
run [`terraform init -upgrade`](https://www.terraform.io/docs/commands/init.html) to download the new version.

For example, given this previous configuration:

```terraform
terraform {
  required_providers {
    akp = {
      source  = "akuity/akp"
      version = "~> 0.10.2"
    }
  }
}

provider "akp" {
  # Configuration options
}
```

Update to the latest 0.11.x version:

```terraform
terraform {
  required_providers {
    akp = {
      source  = "akuity/akp"
      version = "~> 0.11.0"
    }
  }
}

provider "akp" {
  # Configuration options
}
```

## Behavior Changes

### Deleting a Kargo agent that is the default shard agent

In version 0.10.x, deleting a Kargo agent that was currently set as the default shard agent of its parent Kargo instance would cause the provider to silently clear the `default_shard_agent` setting on the instance before deleting the agent. This made it possible to remove an agent that the instance still depended on without an explicit confirmation step.

Version 0.11.0 stops bypassing this validation. Attempting to delete an agent that is the default shard agent now fails with:

```
unable to delete Kargo agent: cannot delete default shard agent, change default shard before deleting
```

If you have Terraform configurations that delete (or replace) a Kargo agent that is the current default shard agent, you must first change the default shard agent on the parent Kargo instance — either by pointing `akp_kargo_default_shard_agent` at a different agent or by removing the `akp_kargo_default_shard_agent` resource entirely — and apply that change before removing the agent.

For example, if you previously had:

```terraform
resource "akp_kargo_agent" "old_agent" {
  # ...
}

resource "akp_kargo_default_shard_agent" "example" {
  kargo_instance_id = akp_kargo_instance.example.id
  agent_id          = akp_kargo_agent.old_agent.id
}
```

And you want to delete `old_agent`, first repoint the default shard agent (or remove it), apply, and then delete `old_agent` in a subsequent apply:

```terraform
resource "akp_kargo_agent" "new_agent" {
  # ...
}

resource "akp_kargo_default_shard_agent" "example" {
  kargo_instance_id = akp_kargo_instance.example.id
  agent_id          = akp_kargo_agent.new_agent.id
}
```

## New Features

### Cross-cluster Secret synchronization

Version 0.11.0 adds a `secrets` block to both `akp_instance` (under `argocd.spec.instance_spec`) and `akp_kargo_instance` (under `kargo.spec.kargo_instance_spec`). It selects which Kubernetes Secrets are synchronized from source clusters to destination clusters managed by the instance.

Secrets opt in by carrying the `akuity.io/secret-sync: "true"` label. Sensitive control-plane Secrets (such as `argocd-secret`) are protected and cannot be synced.

Example for an Argo CD instance:

```terraform
resource "akp_instance" "example" {
  name = "example"
  argocd = {
    spec = {
      instance_spec = {
        secrets = {
          sources = [
            {
              clusters = {
                match_labels = {
                  role = "secret-source"
                }
              }
              secrets = {
                match_expressions = [
                  {
                    key      = "akuity.io/secret-sync"
                    operator = "In"
                    values   = ["true"]
                  }
                ]
              }
            }
          ]
        }
      }
    }
  }
}
```

The same `secrets` block is also accepted on `akp_kargo_instance.kargo.spec.kargo_instance_spec`.

Each entry under `sources` accepts two optional selectors:

- `clusters` — selects the source clusters. If omitted, all clusters are selected.
- `secrets` — selects the Secrets within those clusters. If omitted, all Secrets are selected.

Both selectors support `match_labels` (a map of equality requirements) and `match_expressions` (a list of `{key, operator, values}` requirements). Requirements within and across the two fields are ANDed. A source matches when both the cluster and secret selectors are satisfied.

The corresponding fields are also available on the `akp_instance` and `akp_kargo_instance` data sources.

## Bug Fixes

Versions 0.11.1 and 0.11.2 contain follow-up fixes to the configmap normalization logic introduced earlier in the 0.10.x line:

- **0.11.1** — fixes a regression where normalizing a configmap key caused both the original and the normalized form to be sent to the API, producing `duplicate resources not allowed` errors on `terraform apply`.
- **0.11.2** — additional configmap normalization tweaks and test coverage.

No configuration changes are required to pick up these fixes; upgrade to the latest 0.11.x release available.

## Apply the change

After updating the provider version, optionally adopting the new `secrets` block and addressing any Kargo default-shard-agent deletions in your configuration, run `terraform plan` to verify no unintended changes were introduced. If you notice any unexpected differences, please report them as issues.
